# Security Policy

## Supported versions

Security fixes are provided for the latest released version.

## Reporting a vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use a private reporting channel (for example, GitHub private vulnerability
reporting) and include:

- description of the issue
- impact and attack scenario
- reproduction steps
- potential mitigation, if known

We will acknowledge reports promptly and provide status updates as triage
progresses.

## Disclosure

After a fix is available, we will coordinate disclosure and document the
change in release notes.
